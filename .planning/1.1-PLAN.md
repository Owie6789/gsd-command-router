### TASK-04: Fix Backdrop Scroll

**Dependencies:** None

**Acceptance Criteria:**
- [ ] Scrolling on blurred backdrop does nothing
- [ ] Page behind modal doesn't scroll
- [ ] Scroll events prevented only when modal is open

**Implementation Notes:**
- Add event listener to modal overlay:
  ```javascript
  const modal = document.getElementById('settings-modal');
  modal.addEventListener('wheel', (e) => {
    e.preventDefault();
    e.stopPropagation();
  }, { passive: false });
  
  modal.addEventListener('touchmove', (e) => {
    e.preventDefault();
  }, { passive: false });
  ```
- Alternative: Add `overflow: hidden` to body when modal open:
  ```javascript
  function openSettingsModal() {
    document.body.style.overflow = 'hidden';
    // ... rest of function
  }
  
  function closeSettingsModal() {
    document.body.style.overflow = '';
    // ... rest of function
  }
  ```

**Verification:**
- Open Settings modal
- Scroll mouse wheel on blurred area → page doesn't scroll
- Close modal → scrolling restored

### TASK-05: UI Preferences Tab

**Dependencies:** None

**Acceptance Criteria:**
- [ ] New "UI Preferences" section in Settings modal
- [ ] Font dropdown: Google Sans, Figtree, Poppins, Satoshi
- [ ] Color theme selector: Yellow, Red, Blue, Green, Purple
- [ ] Live preview when selecting
- [ ] Preferences persist in localStorage

**Implementation Notes:**
- Add HTML section in modal:
  ```html
  <div class="settings-section">
    <h3>UI Preferences</h3>
    <div class="preference-row">
      <label>UI Font</label>
      <select id="ui-font-select">
        <option value="Satoshi">Satoshi (Default)</option>
        <option value="Google Sans">Google Sans</option>
        <option value="Figtree">Figtree</option>
        <option value="Poppins">Poppins</option>
      </select>
    </div>
    <div class="preference-row">
      <label>Theme Color</label>
      <div class="color-picker">
        <button class="color-swatch" data-color="yellow" style="background:#e8ff47"></button>
        <button class="color-swatch" data-color="red" style="background:#ff4d4d"></button>
        <button class="color-swatch" data-color="blue" style="background:#4d9fff"></button>
        <button class="color-swatch" data-color="green" style="background:#4dffb0"></button>
        <button class="color-swatch" data-color="purple" style="background:#b57fff"></button>
      </div>
    </div>
  </div>
  ```
- JavaScript to handle changes:
  ```javascript
  // Font change
  document.getElementById('ui-font-select').addEventListener('change', (e) => {
    const font = e.target.value;
    document.documentElement.style.setProperty('--font-family', font);
    localStorage.setItem('gsd:uiFont', font);
  });
  
  // Color change
  document.querySelectorAll('.color-swatch').forEach(swatch => {
    swatch.addEventListener('click', () => {
      const color = swatch.dataset.color;
      // Update CSS variable for accent color
      const colors = {
        yellow: '#e8ff47',
        red: '#ff4d4d',
        blue: '#4d9fff',
        green: '#4dffb0',
        purple: '#b57fff'
      };
      document.documentElement.style.setProperty('--accent', colors[color]);
      localStorage.setItem('gsd:accentColor', colors[color]);
    });
  });
  ```

**Verification:**
- Open Settings → UI Preferences section visible
- Change font → entire UI font updates
- Change color → accent color updates everywhere
- Reload page → preferences persist
