## Phase Details

### Phase 1: Foundation & Search

**Goal:** Users can find the right command faster with better search and filtering

**Depends on:** Nothing (first phase)

**Requirements:** SEARCH-01, SEARCH-02, SEARCH-03, KEYBOARD-01, KEYBOARD-02

**Success Criteria** (what must be TRUE):
  1. User can filter commands by runtime (e.g., "show only Claude Code commands")
  2. User can combine category + runtime filters (e.g., "Core commands for Claude")
  3. User can navigate command list using arrow keys (up/down)
  4. User can expand/collapse command with Enter key
  5. User can clear search with Escape key (single press if focused, double press if not)

**Plans:** TBD

**UI hint:** yes

### Phase 2: State & Persistence

**Goal:** User preferences and activity persist across browser sessions

**Depends on:** Phase 1

**Requirements:** STATE-01, STATE-02, STATE-03, HISTORY-01, FAVORITES-01, FAVORITES-02

**Success Criteria** (what must be TRUE):
  1. User reloads page and returns to last active view automatically
  2. User reloads page and search query + filters are restored
  3. User can view list of recently viewed commands (last 10)
  4. User can mark a command as favorite (star icon)
  5. User can filter to show only favorited commands
  6. User can clear history and clear favorites

**Plans:** TBD

**UI hint:** yes

---

### Phase 3: Export & Sharing

**Goal:** Users can export command references and share specific views

**Depends on:** Phase 1

**Requirements:** EXPORT-01, EXPORT-02, EXPORT-03, SHARE-01, PRINT-01

**Success Criteria** (what must be TRUE):
  1. User can export filtered commands as Markdown file
  2. User can export filtered commands as PDF (via browser print dialog)
  3. User can copy a shareable URL that preserves current filters/search
  4. User opens a shared URL and sees the same filtered view
  5. User prints the page and gets a clean, print-optimized layout (no sidebar, no search box)

**Plans:** TBD

---

### Phase 4: Code Quality

**Goal:** Codebase is maintainable with automated quality checks and tests

**Depends on:** Phase 1

**Requirements:** SPLIT-01, SPLIT-02, LINT-01, TEST-01, TEST-02, CI-01

**Success Criteria** (what must be TRUE):
  1. JavaScript is extracted to `app.js` file (no inline script in HTML)
  2. CSS is extracted to `styles.css` file (no inline style block in HTML)
  3. Running `npm run lint` produces zero errors
  4. Running `npm run test` executes unit tests with >40% coverage
  5. Utility functions (`escHtml`, `getFiltered`, `copyText`) have passing tests
  6. GitHub Actions workflow runs lint + test on every push

**Plans:** TBD

---

### Phase 5: Performance & Polish

**Goal:** Application feels responsive and accessible to all users

**Depends on:** Phase 1

**Requirements:** PERF-01, PERF-02, THEME-01, A11Y-01, A11Y-02

**Success Criteria** (what must be TRUE):
  1. Search input has debouncing (no re-render on every keystroke)
  2. Command list uses virtual scrolling if >100 commands (not applicable yet, but prepared)
  3. User can toggle between dark and light theme
  4. Theme preference persists in localStorage
  5. All interactive elements are keyboard accessible (tab index, focus states)
  6. Color contrast meets WCAG AA standards in both themes

## Dependency Graph

```
Phase 1 (Foundation & Search)
    ↓
Phase 2 (State & Persistence) ← depends on Phase 1 search/filter state
    ↓
Phase 3 (Export & Sharing) ← independent, can run parallel to Phase 2
    ↓
Phase 4 (Code Quality) ← independent, can run anytime
    ↓

```

**Parallel Execution:**
- Phase 2 + Phase 3 can run in parallel
- Phase 4 can run in parallel with any phase
- Phase 5 should follow Phase 1 (search improvements)

## Out of Scope (V2+)

The following are explicitly **not** in this roadmap:

- Backend integration or API server
- User accounts or authentication
- Real-time collaboration features
- AI API integration for live command execution
- Mobile app (React Native, Flutter)
- Desktop app (Electron)
- TypeScript migration
- Visual regression testing
- Analytics dashboard (beyond basic localStorage stats)

These may be added in a future V2 roadmap.
